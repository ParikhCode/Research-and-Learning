*** UART Protocol ***

- Universal Asynchronous Receiver Transmitter
- can be applied to any transmitter and receiver
- cannot use clock signal for communication
- established for serial communication

- two wire are established (transmission and reception)
- Configuration of UART needs to be done before, such as:
	1. Devices with the UART protcol should know the speed of data transmission (Baud Rate)
	2. Configure the data length of both devices
	3. Check data tranmission/ receiving time, forward start bits, and stop bits
- data format
	- sent in frames
	- common config includes: 8 data bits, no parity, 1 stop bit
	- components of UART Frame:
		* Start bit - notfies receiver a packet is arriving
		* Data bits (5-9 bits) - actual data and LSB
		* Parity bit (optional) - error detection; either none, even, or odd number
		* Stop Bit - signal end of frame
	- frame visualized:
		* [IDLE] [START] [D0] [D1] [D2] [D3] [D4] [D5] [D6] [D7] [D8] [Par] [Stop] [Idle]

*** CAN Bus/ LIN

- described as a message-based protocol
- Control Area Network bus 
- common automotive industrial systems
- facilitates efficient communication between sensors and electronic control units (ECUs)
- messages/ frames are received by all devices in the network
	* frames are prioritized by ID numbers
	* types of frames - data, remote, error, adn overload
- the bus (electrical wires) provides 1 point of entry to communciate with all network ECUs
- CAN allows devices in a shared network to communicate with each other
- devices on a bus are called nodes
	* each nodes consists of a CPU, CAN controller and receiver
	* all nodes can send and recieve data, but not at the same time
- data format
	* [SOF] [ID] [RTR] [IDE] [R0] [DLC] [Data Field] [CRC] [ACK] [EOF] [IFS]
	* can not send data directly to a node, but instead onto the network entirely
